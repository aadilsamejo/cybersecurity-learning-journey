# Networking Task 03 - Duplicate IP Address Experiment

## Objective

Understand what happens when two devices are configured with the same IP address.

## Topology

PC1 ----- Switch ----- PC2

## IP Configuration

### PC1

192.168.1.10

### PC2

192.168.1.10

Subnet Mask:

255.255.255.0

## Testing

Attempted communication between the devices using the ping command.

## Results

Packet Tracer did not clearly show an IP conflict in this simple lab environment.

Ping responses were still received.

## Important Learning

In real networks, every device must have a unique IP address.

Duplicate IP addresses can cause:

- Communication problems
- Unstable connectivity
- Network troubleshooting issues

- IP Conflict Concept
